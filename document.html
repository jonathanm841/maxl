Introduction

The technology has been verified by leading experts, and the patents have been prepared by Slingsby Partners LLP, London.

The Technology

The challenge with current VLC decoding practices is the variable nature of each code. GPUs and CPUs struggle with the variable lengths, rarely achieving more than one lookup per clock cycle.

MaxL's beautifully simple, yet revolutionary method is to utilise fixed-bit querying. For example, only 8-bit codes are input into the acceleration table; in practice, this means the table has multiple entries that map to the same decoded symbol.

By returning a decoded symbol immediately from a fixed-bit table lookup and ignoring any trailing bits, the system can deliver multiple decoded symbols in a single clock cycle.

This elegant breakthrough forms the basis of the MaxL system and patents.

Video and Audio Streaming

VLC is deeply integrated into the backbone of modern digital media, serving as a critical compression engine in industry-standard video formats like H.264/MPEG-4 AVC and H.265 High Efficiency Video Coding (HEVC). It also powers the widespread efficiency of MP3 audio compression, demonstrating it's indispensable role in reducing data footprints for high-fidelity content.

Because these legacy standards rely so heavily on these fundamental coding techniques, the MaxL system presents a transformative commercial opportunity:

  • Revamping Legacy Frameworks: By deploying the MaxL system, industries can breathe new performance life into established, aging standards without requiring an overhaul of the underlying encoding methods.
  • Defining New Standards: The superior speed and efficiency of the MaxL architecture provide the necessary foundation for engineers to develop a new, high-performance generation of entropy-encoding standards that surpass the limitations of current techniques.

Autonomous Vehicles

VLC is essential for managing the massive, high-speed data streams generated by sophisticated sensor suites in Autonomous Vehicles.

The primary applications include:

  • Sensor Data Compression: AVs utilise cameras, LiDAR, radar, and ultrasonic sensors that can generate up to 40 Gigabits per second. VLC and entropy coding techniques are applied to these data-intensive modalities to reduce their footprint, which is critical for real-time processing and conserving vehicle energy.
  • GNSS and Telemetry Transmission: To improve communication efficiency between vehicles and infrastructure (V2X), VLC—often via Huffman coding—is used to compress Global Navigation Satellite System (GNSS) logs and other movement-related telemetry. This reduces the data load on communication channels, minimizing delays that are otherwise caused by busy networks.
  • Video Encoding for Connectivity: Similar to standard streaming, AVs employ VLC-based compression (like H.264/AVC) to handle video feeds. This allows vehicles to maintain "network-friendly" and efficient streams for remote assistance, where human operators may need to monitor or intervene in driving scenarios.
